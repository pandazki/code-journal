# Design Context — code-journal

The design direction for code-journal's local web GUI (the journal). The CLI
and headless `sync` path are text-mode — no UI scope here.

## Design Context

### Users

Developers who work with AI coding agents (Claude Code, Codex, Cowork). They run
`code-journal` locally to **look back at their own work** — what they built, how
a project came together day by day, where the intense stretches were.

This is **not** glance-and-go. It is something you *return to*: at the end of a
day, the end of a week, when a project ships. The job-to-be-done is reflection
and continuity, not monitoring. It should make a developer's work feel like it
*amounts to something* — a body of work, not a feed.

### Brand Personality

Three words: **typeset, archival, unhurried.**

code-journal is **a typeset edition of your work** — not a dashboard, a
*publication*. Each day is an entry in a logbook that has been set in proper
type. The aesthetic is editorial print: a literary journal, a printer's
ledger, the back-of-book index of a periodical. Restraint, fine rules,
generous whitespace, the rhythm of a printed page. Calm and considered — it
treats a developer's work as written, edited prose.

### Aesthetic Direction

- **Theme**: both light and dark, via `light-dark()` (default follows system,
  manual toggle). **Light is the home key** — warm cream paper, ink text. Dark
  is its faithful counterpart — warm charcoal "toned paper", warm off-white ink.
  Not "dark mode with glowing accents"; dark is reading at night, not a console.
- **The signature**: the activity heatmap rendered as **ink density on paper**.
  Quiet days are faint, near-empty ledger cells; intense days saturate to deep
  printer's red. It reads as a page someone wrote more and more on — never a
  GitHub-green clone.
- **Typography** (chosen against the reflex — no Fraunces/Newsreader/Lora):
  - Display — **Libre Caslon Display**. High-contrast Caslon; the archival,
    printed-book voice. Masthead, dates-as-headers, day headlines.
  - Body — **Literata**. A literary, screen-tuned serif at reading temperature.
    Narrative prose, day entries, most UI text.
  - Mono — **Spline Sans Mono**. Technical specifics only: file paths, commands,
    commit SHAs, timestamps, the numbers in a byline.
  - No sans-serif. Labels are body serif in tracked uppercase — print, not chrome.
- **Color** (OKLCH; neutrals tinted warm, hue ≈ 60–75):
  - Light — paper `oklch(0.971 0.008 75)`, ink `oklch(0.245 0.018 55)`.
  - Dark — paper `oklch(0.205 0.012 60)`, ink `oklch(0.91 0.014 80)`.
  - Accent — **printer's red**, `oklch(0.52 0.17 30)` light / `oklch(0.63 0.16 33)`
    dark. The 10%: active state, "today", links, the densest heatmap ink. Rare.
- **Layout**: asymmetric, left-aligned. A main reading column (~65ch) with a
  narrow **marginalia** column for a day's stats — printer's notes in the margin.
  No card grids, no centered everything.

### Design Principles

1. **A publication, not a dashboard.** Set the work in type. Numbers are
   fine-print bylines woven into prose — never hero metric cards.
2. **Whitespace and hairlines do the separating.** Reach for a 1px rule or empty
   space before a box, a shadow, or a fill.
3. **Color is rare and earns its place.** A non-paper, non-ink color appears
   only with a reason: today, the active link, the densest ink. Never decoration.
4. **The heatmap is ink, not a scorecard.** Density on a page — the one thing a
   viewer remembers.
5. **Two lenses, equal weight.** The day and the project arc are co-equal; the
   cross-project overview sits above both. No single "main" screen.

### Constraints

- Vanilla HTML + CSS + ES modules, **no build step**, **no runtime npm deps** —
  served as static files from `server/public/`.
- Both themes ship; `prefers-reduced-motion` fully honored.
- One orchestrated page-load stagger; transform/opacity only, exponential easing.

### Anti-references (do not produce)

- A terminal / black developer-tool aesthetic (mono everywhere, glow).
- A KPI analytics dashboard (metric cards + charts).
- A Linear / Notion / SaaS admin panel.
- AI-slop tells: gradient text, glassmorphism, border-left accent stripes,
  card-in-card grids, rounded-rect + drop-shadow, Inter / DM Sans / system UI.
